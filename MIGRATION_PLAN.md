# Migration Plan: Local Development Setup

Step-by-step guide to run BusAcTa Operations locally after cloning from GitHub.

## Prerequisites

- **Node.js** 20+ (required for Vite 7 and TanStack Start)
- **Bun** (package manager — project uses `bun.lock`)
- **Supabase CLI** (optional, for running migrations locally or linking to remote project)
- **Git**

Install Bun (if not installed):

```bash
# Windows (PowerShell)
irm bun.sh/install.ps1 | iex

# macOS/Linux
curl -fsSL https://bun.sh/install | bash
```

## Step 1: Install Dependencies

```bash
cd "E:\Shared Documents\Application Development\BusAcTa Operations"
bun install
```

This reads `package.json` and `bun.lock` to install all dependencies.

## Step 2: Configure Environment Variables

The project already includes a `.env` file with Supabase credentials. If starting fresh, create `.env` in the root:

```env
# Required — Supabase
SUPABASE_URL=https://<project-id>.supabase.co
SUPABASE_PUBLISHABLE_KEY=<anon-jwt-key>
VITE_SUPABASE_URL=https://<project-id>.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=<anon-jwt-key>
VITE_SUPABASE_PROJECT_ID=<project-id>

# Required for SSR / server functions
SUPABASE_SERVICE_ROLE_KEY=<service-role-key>

# Optional — app URLs
APP_ORIGIN=http://localhost:8080
PUBLIC_APP_ORIGIN=http://localhost:8080
PUBLIC_SITE_URL=http://localhost:8080

# Optional — cron protection
CRON_SECRET=<random-secret>

# Optional — integrations (only if using these features)
MS_GRAPH_CLIENT_ID=
MS_GRAPH_CLIENT_SECRET=
MS_GRAPH_REDIRECT_URI=
TWILIO_API_KEY=
TWILIO_FROM_NUMBER=
```

Get Supabase keys from: [Supabase Dashboard](https://supabase.com/dashboard) > Project Settings > API.

## Step 3: Start the Development Server

```bash
bun run dev
```

The dev server starts with HMR (Hot Module Replacement). Open `http://localhost:8080` in your browser.

## Step 4: Connect to Supabase

The app connects to the remote Supabase project using the URL and keys in `.env`. No local Supabase instance is required for development — the remote project handles auth, database, and realtime.

To manage migrations or use Supabase CLI features:

```bash
# Link to the remote project
supabase link --project-ref sgewqhxcknlllpkcurkf

# Check migration status
supabase db diff

# Push local migrations to remote
supabase db push
```

Migrations are in `supabase/migrations/` (160+ files). The remote database is already up to date if synced via Lovable.

## Step 5: Verify the Setup

```bash
# Lint the codebase
bun run lint

# Run unit tests
bun run test

# Validate route links
bun run check:routes
```

Open the app in a browser and verify:

- Login page loads at `/login`
- Auth flow works (login, session persistence)
- Dashboard renders data from Supabase
- Navigation between routes works

## Step 6: Optional — Desktop App

```bash
# Build the Electron desktop app
bun run desktop:build
```

The Electron app loads the web URL (`https://one.busacta.com` by default, or `$BUSACTA_URL` env var). For local development, set `BUSACTA_URL=http://localhost:3000` before building.

Config: `electron/main.cjs`

## Step 7: Optional — Mobile App

```bash
# Sync Capacitor native projects
bun run mobile:sync

# Open in Xcode (macOS only)
bun run mobile:ios

# Open in Android Studio
bun run mobile:android
```

The Capacitor app currently points to the production URL in `capacitor.config.ts`. For local dev, update `server.url` to your local address.

## Step 8: Optional — Backup Worker

The backup worker (`worker/backup-worker/`) is a standalone Node.js service for database backups. It requires:

```env
SUPABASE_URL=https://<project-id>.supabase.co
SUPABASE_SERVICE_ROLE_KEY=<service-role-key>
DATABASE_URL=postgresql://postgres:<password>@db.<project-id>.supabase.co:5432/postgres
PORT=8080
```

## Known Considerations

- **Lovable dependency:** `vite.config.ts` uses `@lovable.dev/vite-tanstack-config`, which bundles TanStack Start, React, Tailwind, and Cloudflare plugins. This works as-is for local development. If full independence from Lovable tooling is needed later, this config would need to be replaced with explicit Vite plugin configuration.
- **Build memory:** Production builds require `--max-old-space-size=8192` (already configured in the `build` script).
- **Generated files:** `src/routeTree.gen.ts` and `src/integrations/supabase/types.ts` are auto-generated. Do not hand-edit — they are regenerated by the TanStack Router plugin and Supabase CLI respectively.
- **Capacitor errors in dev:** You may see `Could not resolve "@capacitor/core"` errors in the dev server console. These are expected — Capacitor packages are for native mobile builds only and don't affect the web dev server.
- **Package manager:** The project uses Bun (`bun.lock`). If using npm as a fallback, pass `--legacy-peer-deps` to handle Tiptap peer dependency conflicts.
